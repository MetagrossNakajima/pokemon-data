## ソースURL

- ベース: `https://projectpokemon.org/images/sprites-models/sv-sprites-home/{4桁番号}.png`
- バリアント: `https://projectpokemon.org/images/sprites-models/sv-sprites-home/{4桁番号}_{連番}.png`

## ファイル名マッピング

- `{4桁番号}.png` → `{ベースポケモン名}.png`
- `{4桁番号}_01.png` → `{ベースポケモン名}_01.png`

## 使用スクリプト

- `script/download_sprites.py`
- Gen 1: `uv run python script/download_sprites.py`
- Gen 2: `python3 -c "from script.download_sprites import download_sprites; download_sprites(gen_start=152, gen_end=251)"`
- Gen 3〜9: `python3 -c "from script.download_sprites import download_sprites; download_sprites(gen_start=252, gen_end=1025)"`

## 既知の課題

- **Tauros（図鑑128）**: `pokemons.json`でのエントリ順が `Tauros-Paldea-Combat` → `Tauros-Paldea-Blaze` → `Tauros-Paldea-Aqua` → `Tauros` のため、ベース名が `Tauros-Paldea-Combat` になっている。ベース画像（`0128.png`）は通常のTaurosであるため、ファイル名を `Tauros.png` にすべきか検討が必要。
- **Wooper（図鑑194）**: 同様に `Wooper-Paldea` が最初のエントリのため、ベース名が `Wooper-Paldea` になっている。ベース画像（`0194.png`）は通常のWooperであるため、ファイル名を `Wooper.png` にすべきか検討が必要。
- **Terapagos（図鑑1024）**: ベース名が `Terapagos-Normal` になっている。`Terapagos` にすべきか検討が必要。
